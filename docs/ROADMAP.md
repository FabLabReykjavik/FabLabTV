# Roadmap

## Good next improvements

- Optional authentication for `/remote` on public networks.
- More robust Global Pulse parsers as public Fab ecosystem sites evolve.
- Better Fab Academy highlight metadata and thumbnails.
- Local Pulse scheduling windows for messages and workshops.
- Import/export settings for easier deployment across multiple labs.
- Optional machine usage and visitor statistics adapters.
- Optional calendar integration for local workshops.

## Design principles

- Keep the TV screen stable and kiosk-friendly.
- Keep Remote simple enough for non-technical staff.
- Prefer file-based configuration over databases unless the project clearly needs one later.
- Avoid hardcoded future event content; fetch and cache global data instead.
